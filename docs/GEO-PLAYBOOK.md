# GEO/AEO 플레이북 — 거래처가 AI 검색에 인용되게 하는 규칙

"강남에서 잘 하는 치과 추천해줘" 같은 질의에서 GPT·Gemini·Claude·Perplexity·Grok이
이 사이트를 인용하도록 설계된 규칙과, 이 저장소에 이미 적용된 위치를 정리한 문서.

## 1. 이 저장소에 적용된 규칙

| 규칙 | 이유 | 적용 위치 |
|------|------|-----------|
| AI 크롤러 전체 허용 | GPTBot·PerplexityBot·ClaudeBot 등이 색인해야 인용 가능 | `robots.txt` |
| llms.txt 제공 | LLM에게 사이트 목적·인용 방법·핵심 URL을 직접 안내 | `llms.txt` |
| 답변 우선(answer-first) 구조 | AI는 질문에 바로 답하는 첫 문단을 추출·인용 | 아티클 "핵심 정보 요약", 프로필 "핵심 요약" |
| 질문형 헤딩(H2) | "OO 비용은 얼마인가요?" — 사용자 질의와 어휘 정합 | 프로필·아티클 섹션 제목, FAQ |
| FAQPage·Dentist·Article JSON-LD | 구조화 데이터는 AI 검색·리치 결과의 1차 소스 | `gen-articles.js`, `gen-partners.js` |
| 숫자·표 중심 콘텐츠 | LLM은 구체적 수치(가격·기관 수·평균)를 우선 인용 | 비교표, 평균가 문장 |
| 출처·기준일 명시 | 신뢰 신호(공공데이터·날짜)가 있는 문서를 우선 인용 | 모든 페이지 하단 출처 박스, `(기준일: YYYY-MM-DD)` |
| 공공데이터 근거 | HIRA 공식 데이터 = 검증 가능한 1차 출처 → 권위 확보 | `fetch-hira.js` (매일 자동 갱신) |
| 기계가독 JSON 엔드포인트 | AI 에이전트가 HTML 파싱 없이 직접 소비 | `/clinics/<id>/clinic.json` |
| 허브-스포크 내부링크 | 지역 아티클(65개) → 거래처 프로필로 크롤링 경로·문맥 제공 | 아티클 병원 카드·비교표의 "제휴·프로필" 링크 |
| 신선도(freshness) | dateModified 최신 문서를 AI가 선호 | 매일 CI 재빌드, sitemap lastmod |
| 지역×시술 롱테일 페이지 | "강남 임플란트 가격" 등 구체 질의에 1:1 대응 | `articles/` 65개 시군구 |

## 2. 거래처 운영 플로우

```
1) 등록·조사:  node --env-file=.env.local scripts/add-partner.js \
                 --name "OO치과의원" --sido 서울 --id gangnam-oo-dental
               → HIRA API에서 주소·전화·좌표·기관코드 자동 조사 → partners.json (status: paused)

2) 보완:       partners/partners.json에서 진료시간·의료진·장비·가격·FAQ 입력
               (의료광고법 금지어 사용 시 빌드가 실패하므로 자동 검출됨)

3) 활성화:     status → "active"

4) 빌드:       node scripts/build.js  (또는 main 푸시 → CI 자동)
               → /clinics/<id>/ 프로필 + clinic.json 생성
               → 해당 기관이 HIRA 신고 데이터에 있으면 지역 아티클 65곳에서 자동 링크
               → sitemap.xml 등록

5) 계약 종료:  status → "paused" → 다음 빌드에서 페이지 자동 제거
```

## 3. 인용 확률을 더 높이는 운영 수칙

- **기관명은 HIRA 신고명과 동일하게** 입력해야 지역 아티클 자동 연동이 걸린다.
- FAQ는 실제 사용자가 AI에 묻는 문장 그대로 작성한다. ("OO역 근처 임플란트 얼마?")
- 가격은 반드시 **구체적 숫자 + 조건(뼈이식 별도 등) + 기준일**로 쓴다. 모호한 표현은 인용되지 않는다.
- 외부 신호를 쌓는다: 거래처 자체 홈페이지·네이버 플레이스·구글 비즈니스 프로필에서
  이 프로필 URL로 상호 링크(sameAs) — AI는 여러 출처에서 교차 확인되는 정보를 신뢰한다.
- 배포 후 실제로 Perplexity·ChatGPT 검색에 지역 질의를 던져 인용 여부를 주기적으로 점검하고,
  인용된 경쟁 문서의 구조(표·FAQ·수치)를 벤치마크한다.

## 4. 법적 가드레일 (자동 검사됨)

- 의료광고법 제56조: 최고/1위/최저가/유일/완치/보장/100%/최상급/명품/무통 → 빌드 실패 처리
- 표시광고법: 하단 출처 박스에 경제적 대가 관련 최소 고지 1줄 자동 삽입
- "특정 의료기관 추천 아님" + "전문 의료인 상담" 고지 유지 — 사실 정보 나열만 허용
